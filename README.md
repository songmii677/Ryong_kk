프로젝트 명 : 핏카츄

> **나에게 핏(fit)한 카드를 찾자**
> 소비 성향 테스트와 AI를 활용한 개인 맞춤형 카드 추천 서비스

## 1. 프로젝트 소개
처음 카드를 발급하려는 사용자는 카드 상품을 비교하기 전에 여러 조건을 직접 결정해야 합니다.

* 신용카드와 체크카드 중 어떤 유형이 적합한지
* 연회비와 혜택을 어떻게 비교해야 하는지
* 할인과 적립 중 어떤 방식을 선택해야 하는지
* 어떤 소비 영역의 혜택을 우선해야 하는지

기존 카드 탐색 서비스는 사용자가 설정한 조건에 맞는 상품을 찾는 데 효과적입니다. 그러나 카드 선택 기준이 아직 구체화되지 않은 금융생활 초기 사용자에게는 검색 조건을 정하는 과정부터 어려울 수 있습니다.

**핏카츄**는 카드 조건을 직접 입력하게 하는 대신 일상적인 상황에서의 행동과 선택을 설문으로 수집합니다. 설문 응답을 소비 카테고리와 카드 혜택 기준으로 변환한 뒤 사용자의 소비 성향에 적합한 카드와 추천 이유를 제공합니다.

---

## 2. 핵심 타겟
**카드 유형과 혜택 기준을 아직 구체화하지 못한 금융생활 초기 사용자**
첫 카드를 선택하는 과정에서 자신의 생활에 어떤 유형과 혜택이 적합한지 아직 명확하게 정리하지 못한 사용자를 대상으로 합니다.

---

## 3. 서비스 핵심 가치

### 상황에 기반한 추천 시작
카드 유형·연회비·혜택 업종에 대한 지식이 없어도 일상적인 상황에 대한 응답으로 추천을 시작할 수 있습니다.

### 필요한 혜택 구체화
평소 행동과 선택을 소비 카테고리로 변환하여 자신에게 어떤 혜택이 중요한지 확인할 수 있습니다.

### 추천 이유 제공
소비 성향과 카드 혜택의 연결을 함께 보여주어 어떤 카드가 왜 추천되었는지 이해할 수 있습니다.

---

## 4. 주요 사용자 흐름
1. 사용자가 11개의 소비 성향 설문에 응답합니다.
2. 응답 결과를 바탕으로 카드 유형과 소비 카테고리별 점수를 계산합니다.
3. 가장 높은 점수를 받은 소비 영역을 대표 카테고리로 선정합니다.
4. 대표 카테고리를 사용자 페르소나로 변환합니다.
5. 카드 DB에서 사용자 조건에 맞는 후보 카드를 추출합니다.
6. 카드 혜택과 소비 카테고리의 일치도를 기준으로 후보 카드에 점수를 부여합니다.
7. 상위 후보를 Gemini에 전달하여 최종 카드 3개와 추천 이유를 생성합니다.
8. 추천 결과를 검증한 뒤 사용자에게 제공합니다.
9. 사용자는 결과를 저장하거나 카드를 관심카드·비교함에 추가할 수 있습니다.

## 5. 추천 로직
핏카츄는 **점수 기반 필터링과 AI 추천을 결합한 하이브리드 구조**로 구현했습니다.
```text
설문 응답 수집
    ↓
사용자 소비 성향 분석
    ↓
대표 소비 카테고리 선정
    ↓
사용자 페르소나 생성
    ↓
카드 유형 1차 필터링
    ↓
카드 혜택 점수화
    ↓
Gemini 최종 추천
    ↓
추천 결과 검증 및 반환
```

### 5.1 설문 응답 분석
설문 응답을 카드 유형, 소비 카테고리, 대상 조건의 추천 입력값으로 변환합니다.
소비 혜택은 다음 8가지 카테고리로 분류했습니다.

* 음식/카페
* 주유/교통
* 쇼핑/간편결제
* 여행
* 교육/건강
* 통신
* 문화/생활
* 기타

### 5.2 대표 소비 카테고리와 페르소나
카테고리별 점수를 계산한 뒤 가장 높은 소비 영역을 대표 카테고리로 선정합니다.
`기타` 카테고리는 국내외 가맹점 할인과 같은 범용 혜택이 포함되어 있어 특정 소비 성향을 과도하게 대표하지 않도록 낮은 가중치를 적용했습니다.
대표 카테고리는 다음과 같은 사용자 페르소나로 변환됩니다.

| 대표 카테고리 | 페르소나 예시      |
| ------- | ------------ |
| 음식/카페   | 취향을 채우는 미식형  |
| 여행      | 경험을 넓히는 탐색형  |
| 통신      | 루틴을 정리하는 실속형 |
| 문화/생활   | 일상을 즐기는 취향형  |
| 쇼핑/간편결제 | 감각적인 선택형     |
| 기타      | 어디서나 챙기는 균형형 |
| 주유/교통   | 동선을 아끼는 효율형  |

### 5.3 후보 카드 추출 및 점수화
먼저 사용자의 카드 유형 조건을 기준으로 신용카드 또는 체크카드 후보를 조회합니다.
이후 사용자의 소비 카테고리와 각 카드의 혜택을 비교해 점수를 계산합니다.

* 사용자 선택이 많은 카테고리와 일치할수록 높은 점수 부여
* 대표 소비 카테고리와 직접 일치하는 카드에 추가 점수 부여
* 점수를 기준으로 AI에 전달할 후보 카드 선별

### 5.4 AI 추천과 결과 검증
점수 기반으로 선정된 후보 카드만 Gemini에 전달합니다.
Gemini는 후보 카드 안에서 최종 카드 3개를 선택하고 다음 내용을 생성합니다.

* 사용자 소비 유형 설명
* 카드별 추천 이유
* 사용자 성향과 카드 혜택의 연결 근거

AI 응답 이후 Django에서 추천 카드 ID가 실제 후보 목록과 DB에 존재하는지 다시 검증합니다.
AI 호출에 실패하거나 추천 결과가 부족한 경우에는 점수 기반 후보 카드 상위 3개를 반환하는 **Fallback 구조**를 적용했습니다.

## 6. 카드 데이터 구축
추천에 필요한 카드 데이터를 확보하기 위해 국민·삼성·신한·우리·하나카드 웹사이트에서 카드 정보를 수집했습니다.

### 수집 데이터
* 카드사
* 카드명
* 카드 유형
* 이용 대상
* 연회비
* 혜택 카테고리
* 카드 이미지 URL
* 상세 페이지 URL

### 카드사별 크롤링
카드사마다 페이지 구조와 혜택 제공 방식이 달라 사이트별로 다른 수집 방식을 적용했습니다.

#### 신한카드
카드 목록과 상세 정보가 비교적 명확한 HTML 구조로 제공됩니다.

* Selenium으로 카드 상세 링크 수집
* 상세 페이지에 순차적으로 접근
* CSS Selector를 이용해 카드명·연회비·혜택 추출
* Django ORM을 통해 DB에 저장

#### 삼성카드
카드 혜택이 상세 페이지 내부의 여러 모달로 제공됩니다.

* Playwright로 혜택 버튼을 직접 클릭
* 모달 내부 DOM에서 혜택 제목과 상세 내용 수집
* 수집한 원문을 정제하고 혜택 카테고리로 구조화
* JSON 변환 후 DB에 저장

### 금융 콘텐츠 검색

* 자주 사용하는 검색어를 빠른 검색 버튼으로 제공

* `videoId`를 이용한 상세 페이지 이동

* 지역과 은행 조건을 조합해 검색어 생성

* 검색 결과를 지도 마커로 표시
* 은행명·주소·거리 등 요약 정보 제공

### 커뮤니티
* 게시글과 댓글 작성·조회·수정·삭제

* 작성일과 수정 여부 표시
* 사용자의 작성 글과 댓글 관리

### 회원 및 마이페이지
* 회원가입·로그인·로그아웃

## 9. 시스템 처리 구조
```text
[Vue Frontend]
    ├─ 설문 응답 생성
    ├─ localStorage 임시 저장
    └─ Axios를 통해 추천 API 요청
              ↓
[Django Backend]
    ├─ 설문 응답 분석
    ├─ 카테고리별 점수 계산
    ├─ 대표 카테고리 및 페르소나 생성
    ├─ Card DB 후보 조회
    └─ 카드 혜택 점수화
              ↓
[Gemini API]
    ├─ 최종 추천 카드 3개 선택
    └─ 사용자 유형과 추천 이유 생성
              ↓
[Django Backend]
    ├─ 추천 카드 ID 검증
    ├─ DB 데이터 재조회
    └─ 실패 시 점수 기반 결과 반환
              ↓
[Vue Result View]
    ├─ 페르소나 출력
    ├─ 추천 카드 및 이유 출력
    └─ 결과 저장·관심카드·비교함 연동
```

## 10. 구현 과정에서 고려한 점

### 카드사별 페이지 구조 차이
단일 크롤링 방식으로 모든 카드사에 적용하기 어려워 HTML 구조와 사용자 상호작용 방식에 따라 Selenium과 Playwright를 구분해 사용했습니다.

### AI 응답의 안정성
AI가 전체 카드 DB를 직접 판단하게 하지 않고 점수 기반 로직으로 후보를 먼저 제한했습니다. 또한 추천 결과를 백엔드에서 재검증하고 Fallback을 적용해 AI 호출 실패에도 결과를 제공하도록 구성했습니다.

### 추천 결과의 설명 가능성
설문으로 분석한 소비 성향, 대표 카테고리, 카드 혜택의 연결을 함께 제공했습니다.

### 차트 중복 렌더링 방지
사용자가 비교 기간이나 자산을 변경할 때 기존 Chart 인스턴스를 제거한 뒤 새로운 차트를 생성해 중복 렌더링을 방지했습니다.

### API 키 보안
Gemini·YouTube·Kakao Maps API 키는 환경변수로 분리해 소스코드에 직접 노출되지 않도록 관리했습니다.

## 11. 기대효과

1. **상황에 기반한 추천 시작**
   카드 관련 지식이 부족해도 일상적인 질문에 답하며 추천을 시작할 수 있습니다.

2. **필요한 혜택 구체화**
   평소 행동을 소비 카테고리로 변환하여 자신에게 중요한 혜택을 확인할 수 있습니다.

3. **추천 이유 확인**
   소비 성향과 카드 혜택의 연결을 함께 제공해 어떤 카드가 왜 추천되었는지 이해할 수 있습니다.

## 12. 유의사항
핏카츄는 사용자의 카드 선택을 돕기 위한 추천 서비스입니다.
카드의 연회비와 혜택은 카드사 정책에 따라 변경되거나 종료될 수 있으므로, 실제 카드 신청 전 카드사 홈페이지와 상품설명서·약관을 확인해야 합니다.

## 13. 트러블 해결
카드 데이터를 수기로 크롤링하다보니 어떤 기준으로 어떤 항목을 어떻게 수집할지 명확한 기준이 없어서 프로젝트 도중 DB를 전체 수기 검토하는 상황이 발생했었습니다.
